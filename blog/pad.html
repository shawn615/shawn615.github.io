The Core Idea

PAD is best understood as a direct answer to a specific weakness in image-to-3D generation: many methods generate an object in a canonical coordinate system and then try to rotate, translate, or optimize it back into the observed pose. That separation is convenient for training, but it creates a fragile interface. The generated object may be semantically plausible, yet still fail to sit exactly where the input image says it should.

The paper's thesis is that pose alignment should not be a postprocess. PAD generates the object directly in observation space by using the input image's estimated metric geometry as a 3D anchor. A monocular depth model produces a visible partial point cloud; a vecset VAE encodes that partial point cloud into geometric latent tokens; the diffusion transformer receives those tokens by direct sequence concatenation while denoising the target 3D latent.

Core idea: replace "generate canonical object, then estimate pose" with "generate the completed object in the camera-aligned coordinate system from the start." The visible geometry is treated as a scaffold, not merely as an image feature.

This paper sits close to Pixal3D in spirit, but the mechanism is different. Pixal3D emphasizes pixel-to-voxel feature lifting; PAD emphasizes partial point-cloud conditioning in a native 3D diffusion latent. Both papers are part of the same broader trend: high-fidelity image-to-3D is becoming less about unconstrained hallucination and more about respecting geometric evidence.

Problem Setting

PAD targets single-image posed 3D generation. Given an image of an object, the goal is not only to produce a high-quality 3D shape, but to produce it in the same pose and spatial frame as the observation. This matters whenever the asset must be placed back into a scene: robotic simulation, AR, gaming, scene reconstruction, or object-level editing inside an existing camera view.

The hard part is that canonical-space training and observation-space deployment pull in different directions. A canonical latent distribution is easier to model because all objects share a normalized orientation and scale. But the input image is not canonical. It is a particular view, often with symmetry, occlusion, truncation, and ambiguous rotation. Once the generator and pose estimator are decoupled, a small error in either part can become a visible spatial mismatch.

PAD assumes a monocular depth estimator can provide a useful, though noisy, visible 3D scaffold. The method does not require perfect depth. In fact, one of the paper's important training decisions is to inject noisy estimated depth during training so the model does not become a brittle autoencoder of ideal point clouds.

Background

Canonical-Then-Rotate Pipelines

A typical compositional image-to-3D pipeline first generates a canonical object and then estimates a rigid transform to place it in the scene. This is modular, but it loses a direct link between input pixels and final 3D coordinates. Symmetric objects are especially dangerous because several rotations can explain the same image-level semantics while only one is spatially correct.

Vecset VAEs

PAD uses a vecset VAE, a representation family common in recent 3D generative models. The encoder maps a shape to a set of latent tokens, and the decoder predicts an SDF value at queried 3D coordinates. Because the latent is treated as a set, token order is not the central structure; the geometry is represented by the collection of structural features. Marching Cubes then extracts the zero level set as a mesh.

Flow Matching for 3D Latents

The generative backbone follows a flow-matching formulation. Rather than learning a reverse diffusion process in pixel space, PAD adapts a DiT-style velocity estimator over 3D latent tokens. This is inherited from the Hunyuan3D-2.1 family, which gives PAD a strong pretrained 3D prior before pose-aware finetuning.

Monocular Depth as a 3D Anchor

A depth map is still an image-space signal. PAD lifts it into 3D by unprojection, turning visible pixels into a partial point cloud. This is the paper's key representational choice: the condition should live in 3D if the model must reason about 3D pose and spatial layout.

Method

1. Generate in Observation Space

The paper explicitly contrasts two mappings. Canonical pipelines learn a canonical mesh distribution \(p(M \mid I)\) and a pose distribution \(p(T \mid I)\), then form \(\hat{M}=T\circ M\). PAD instead tries to learn \(p(\hat{M}\mid I)\) directly, where \(\hat{M}\) already lives in the input camera-aligned frame.

This design removes an entire failure mode. If the object is symmetric, transparent, occluded, or locally ambiguous, a separate pose module can make a visually plausible but wrong decision. PAD still has to complete unseen geometry, but it does not have to re-register a generated canonical shape back to the observation.

2. Unproject Depth to a Partial Point Cloud

Given an input image \(I\), PAD uses MoGe to estimate metric depth and camera intrinsics. The visible surface is then lifted to a partial point cloud. In simplified notation:

\[ P_{\text{partial}} = \left\{ K^{-1} \begin{bmatrix} u \\ v \\ 1 \end{bmatrix} D(u,v) \; \middle|\; (u,v)\in I \right\}. \]

This is not a complete object. It is the observed surface scaffold. The generative model is still responsible for completing the back side, resolving occlusions, and producing watertight geometry.

3. Encode the Point Cloud as Latent Tokens

PAD maps the partial point cloud into the same kind of latent space used by the shape generator:

\[ z_{\text{geo}} = \mathcal{E}(P_{\text{partial}}). \]

Before encoding, the point cloud is padded or cropped to a fixed size. The paper reports randomly downsampling or duplicating partial point clouds to 81,920 points to match the shape encoder's input specification.

4. Concatenate Geometry into the Diffusion Transformer

The geometric latent is not used as a vague auxiliary embedding. PAD directly concatenates \(z_{\text{geo}}\) with the noisy target latent \(z_t\) along the sequence dimension:

\[ v_{\theta}(z_t,t,c) := v_{\theta} \left( \operatorname{Concat}(z_t,z_{\text{geo}}), t, c_{\text{img}} \right). \]

The image embedding \(c_{\text{img}}\) still supplies global semantic information, but local geometric anchoring comes from the encoded partial 3D evidence. This is a stronger constraint than asking cross-attention over a depth map to implicitly learn 3D spatial relations.

5. Stabilize the Domain Shift

PAD initializes from Hunyuan3D-2.1, which is a strong canonical 3D prior. Moving it into observation space is not a small finetune. The paper addresses this with curated paired data from Objaverse and 3D-Front, 24 rendered views per object, view-aligned ground-truth meshes, and back-projected depth maps.

A key engineering detail is depth noise augmentation. If the model only sees perfect training-time partial point clouds, it overfits to them and fails when MoGe produces noisy inference-time geometry. PAD randomly replaces ground-truth point clouds with estimated-depth point clouds and injects point noise, forcing the generator to treat the condition as structural guidance rather than an exact answer.

The important move is not simply using depth. It is representing depth-derived evidence as 3D latent tokens that the generator can condition on natively.

Compositional Scene Generation

PAD extends naturally to scenes because each generated object already lives in the observation coordinate frame. The scene pipeline estimates a global partial point cloud from the input image, segments object instances, extracts each object's partial point cloud and crop, generates each object independently, and then takes a union:

\[ M_{\text{scene}} = \bigcup_i M^{(i)}. \]

This is the main practical payoff of observation-space generation. The method does not have to solve a separate per-object 7-DoF registration problem after generation. Of course, the decomposition still depends on segmentation and depth quality, but the final assembly is less brittle than canonical object generation plus pose optimization.

Experiments

Training Setup

PAD finetunes Hunyuan3D-2.1 with AdamW, peak learning rate \(1\times10^{-5}\), batch size 16 on 8 NVIDIA A100 GPUs, 50k iterations, and a reported training time of about two days. The authors use CFG drop rate 0.1 during training and 50 inference steps with CFG strength 3.0.

The curated training set comes mainly from Objaverse and 3D-Front. The appendix says the filtering pipeline yields approximately 20k high-quality objects after removing low-quality meshes, trivial geometry, multi-object clusters, and objects whose VAE reconstruction suggests poor watertightness. The scene robustness story is strengthened by synthetic occlusion augmentation.

Posed Object Generation

The object-level evaluation uses about 1,000 held-out GSO objects. For each object, the authors render a single RGBA image from a random viewpoint and transform the ground-truth mesh into the camera coordinate system. That evaluation design is important: it measures alignment in the observed frame rather than after forgiving registration.

Ablations

The ablations are particularly useful because they test the paper's core mechanism. Removing the 3D condition causes major structure and alignment failures. Replacing the point-based representation with a 2D depth-map condition is also worse, even though the two carry related information. The representation format matters: lifted 3D evidence is easier for a 3D generator to use than image-space depth.

The depth-noise ablation is also important. Without noisy conditioning during training, the model becomes overly dependent on clean point clouds and fails under realistic depth-estimator errors. This is a practical reminder: geometry conditioning is powerful, but it must be trained as a noisy scaffold, not an oracle.

Strengths

Right failure mode

The paper identifies pose mismatch as a structural flaw in canonical-then-rotate pipelines, not just an evaluation nuisance.

Native 3D conditioning

Depth is lifted to a partial point cloud and encoded as 3D latent tokens, giving the generator a geometric anchor in its own domain.

Pragmatic reuse

PAD starts from Hunyuan3D-2.1 rather than training a full 3D generator from scratch, making the contribution focused.

Scene extension

Observation-space generation naturally supports object-wise scene composition with simple union rather than pose registration.

Limitations

The first limitation is upstream dependence. PAD's spatial accuracy depends on monocular depth and segmentation quality. If MoGe gives a distorted scale, wrong boundary, or flying points, those errors can propagate into the generated mesh. The depth-noise training strategy mitigates this, but it does not remove the dependency.

Second, occlusion remains ambiguous. PAD can complete hidden geometry using a learned prior, but a heavily occluded object may have multiple plausible completions. Observation-space anchoring helps with visible surfaces and layout; it does not make unseen structure uniquely determined.

Third, the method focuses mainly on geometry. The appendix notes that texture synthesis is handled using Hunyuan3D-Paint after shape generation. That is a reasonable engineering choice, but it means the core method is not yet a unified geometry-material generator.

Finally, the code release is useful but not complete: the README lists scene generation model weights, training code, and data preprocessing code as future release items. That matters for reproducibility, especially because the data curation and training curriculum are central to the paper's performance.

Research Insight

Research insight: PAD and Pixal3D point to the same bottleneck from different angles: high-fidelity image-to-3D requires a hard geometric contract between the observation and the generated 3D representation.

The deeper lesson is that canonical object priors and observation-space constraints should probably coexist. Canonical spaces are efficient for learning object categories, symmetries, and reusable asset structure. Observation spaces are better for matching a particular image, a particular camera, and a particular scene layout. PAD shows one way to move a pretrained canonical generator toward observation-space fidelity by anchoring it with a partial point cloud.

The method is also a useful reminder that "use depth" is too vague. Depth can be an image channel, a cross-attention token, a point cloud, a signed-distance constraint, a differentiable rendering loss, or a latent sequence. PAD's ablation suggests that the representation of geometry matters almost as much as the presence of geometry.

Practical Perspective

For research engineers, PAD is attractive if the target workflow cares about putting generated assets back into the original scene. Robotics, simulation, AR, and object editing all need coordinate consistency. A canonical mesh that looks good but needs delicate registration is often less useful than a slightly constrained mesh that already lands in the right frame.

The practical cost is pipeline complexity. PAD needs monocular depth, object masks or crops, a Hunyuan3D-style 3D generator, a finetuned denoiser, and post-generation texturing. This is not a small lightweight module. The value is in reducing the downstream pose-registration burden.

Future Directions

  • Uncertainty-aware depth conditioning: let the model know which parts of the partial point cloud are reliable instead of treating all points similarly.
  • Joint geometry and material generation: bring texture and material synthesis into the same observation-space latent process.
  • Canonical-observation dual latents: preserve editable object-centric structure while maintaining observation-space alignment.
  • Better occlusion modeling: explicitly represent visible, occluded, and uncertain regions rather than using a single partial point-cloud scaffold.
  • Full scene release and evaluation: scene generation code, weights, and preprocessing would make the compositional claims easier to reproduce.
  • Interactive correction: allow users to edit depth, masks, or partial geometry before generation to control alignment failures.

Key Takeaways

  • PAD treats pose as part of generation. It does not generate a canonical object and ask another module to align it later.
  • The partial point cloud is the anchor. Monocular depth is lifted into 3D, encoded into latent tokens, and concatenated into the diffusion transformer.
  • Noise augmentation is not optional. Without it, the model overfits to clean training-time geometry and fails under realistic estimated depth.
  • The scene story is a natural consequence. If each object is generated in the observed frame, composition becomes a union rather than a registration problem.
  • The next challenge is richer assets. Observation-space geometry is useful, but production workflows also need materials, parts, physical affordances, and editability.
