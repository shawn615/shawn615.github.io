The Missed Opportunity in 3DGS Compression

Most 3DGS compression methods treat each Gaussian (or anchor) as an independent coding unit. Prune some, quantize the rest, entropy-code them in one shot—done. But if you look at the anchor features of a typical scene, neighboring anchors are highly correlated. The paper shows cosine similarities of 0.6–0.9 between adjacent anchors. That’s a massive pool of spatial redundancy just sitting there, untouched.

Image compression figured this out decades ago: context models predict the distribution of the next pixel/latent from already-decoded neighbors, dramatically reducing entropy. ContextGS brings this idea to 3DGS for the first time, but at the anchor level rather than the pixel level. The result: an autoregressive coding scheme that squeezes out the spatial redundancy that previous methods leave on the table.

The context model analogy: image compression predicts pixels from decoded neighbors in raster order. ContextGS predicts anchor features from decoded coarser-level anchors in 3D space—but only needs 3 levels, not millions of steps.

Background: Scaffold-GS Anchors

ContextGS builds on Scaffold-GS, so it’s worth recapping the data structure. Instead of storing every Gaussian’s attributes individually (the vanilla 3DGS way), Scaffold-GS introduces anchor points—voxel representatives that store a compact feature vector \(\mathbf{f} \in \mathbb{R}^{32}\), position \(\mathbf{x}\), scaling \(\mathbf{l}\), and offsets \(\mathbf{O}\). Small MLPs decode the full Gaussian attributes (color, rotation, scale, opacity) from each anchor at render time, conditioned on view direction.

This already reduces redundancy a lot—nearby Gaussians share an anchor feature. But Scaffold-GS still stores each anchor independently. If two neighboring anchors represent similar geometry (e.g., a flat wall), their features are correlated but nobody exploits that correlation. ContextGS does.

The Core Idea: Hierarchical Anchor Partitioning + Autoregressive Coding

The method has two main ingredients: (1) divide anchors into hierarchical levels, and (2) use already-decoded coarser levels to predict the distribution of finer levels for entropy coding.

1. Anchor Partitioning

Given all \(N\) anchors, partition them into \(K=3\) disjoint levels: \(\mathbf{V} = \mathbf{V}_0 \cup \mathbf{V}_1 \cup \mathbf{V}_2\). Level 2 is coarsest (fewest anchors), level 0 is finest (most anchors). The division uses a bottom-up voxelization: quantize anchor positions at increasing voxel sizes \(\epsilon_k = \kappa^k \cdot \epsilon\), and pick a representative anchor per voxel for each level.

The target ratio between adjacent levels is \(\tau = |\mathbf{V}_{k+1}| / |\mathbf{V}_k| \approx 0.2\), found via binary search on \(\kappa\). So roughly: Level 2 has 4% of anchors, Level 1 has 16%, Level 0 has 80%.

Anchor partitioning: Level 2 (coarsest, ~4%) is encoded first, then Level 1 using L2 as context, then Level 0. The target ratio τ ≈ 0.2 between adjacent levels is found automatically via binary search on κ.

2. Autoregressive Coding

Encoding proceeds coarse-to-fine: Level 2 first, then Level 1 conditioned on decoded Level 2, then Level 0 conditioned on both. For each anchor \(\mathbf{v}_i^k\) in level \(k\), its distribution is modeled as:

\[ p(\mathbf{f}_i^k \mid \boldsymbol{\psi}_i^k) = \left(\mathcal{N}(\boldsymbol{\mu}_i^k, \boldsymbol{\sigma}_i^k) * \mathcal{U}\left(-\tfrac{\Delta}{2}, \tfrac{\Delta}{2}\right)\right)(\mathbf{f}_i^k) \]

where \(\boldsymbol{\mu}, \boldsymbol{\sigma}, \Delta\) are predicted by a small MLP from the context \(\boldsymbol{\psi}_i^k\). The context includes:

  • For the coarsest level (L2): just the anchor’s position \(\mathbf{x}_i\) (no decoded neighbors yet)
  • For finer levels (L1, L0): the already-decoded neighboring anchor’s feature \(\mathbf{f}_j^{k+1}\), scaling \(\mathbf{l}_j^{k+1}\), and position \(\mathbf{x}_i^k\)

The key insight: this is not per-element autoregressive (which would be agonizingly slow). It’s per-level—only 3 iterations of the loop, and within each level all anchors are coded in parallel. This is why the decoding overhead is negligible compared to pixel-level autoregressive models in image compression.

3. Anchor Forward

A subtle but important trick: anchors decoded at coarser levels are reused at the finest level for rendering. They’re not just context—they’re actual scene content. This avoids storing duplicate anchors across levels and slightly reduces the total anchor count.

Hyperprior: Giving L2 Something to Work With

The coarsest level (L2) has no decoded neighbors to condition on. Position alone isn’t expressive enough to predict feature distributions accurately. The solution: attach a small hyperprior feature \(\mathbf{z}_i \in \mathbb{R}^{50/h_c}\) to each anchor (where \(h_c = 4\), so 12–13 dims). These are quantized to integers and entropy-coded using a fully factorized prior model à la Ballé et al.

The hyperprior is jointly optimized end-to-end: it learns to capture whatever information about the anchor feature distribution can’t be predicted from position and spatial context alone. In practice, it adds a small storage overhead (~1 MB on BungeeNeRF) but enables much tighter entropy bounds.

Storage breakdown for a ContextGS model. Features dominate at 41%, but everything is entropy-coded. Hyperprior adds only 6% overhead for significant coding gains.

Training Objective

The loss is a standard rate-distortion tradeoff, jointly optimizing rendering quality and bitrate:

\[ \mathcal{L} = \mathcal{L}_\text{scaffold} + \lambda_e \mathcal{L}_\text{entropy} + \lambda_m \mathcal{L}_\text{mask} \]

where \(\mathcal{L}_\text{scaffold}\) is the standard Scaffold-GS reconstruction loss (L1 + SSIM), \(\mathcal{L}_\text{entropy}\) is the total bitrate estimate (hyperprior bits + anchor feature bits across all levels), and \(\mathcal{L}_\text{mask}\) regularizes the offset masks. Different \(\lambda_e\) values give different rate-distortion tradeoffs. The training uses the same 30K iterations and growing hyperparameters as Scaffold-GS.

At test time, rendering is exactly Scaffold-GS—no context model, no MLP overhead. The entropy model is only used for encoding/decoding the bitstream. Once decoded, the anchor features are identical to what Scaffold-GS would use.

A few observations:

  • Just adding entropy coding + masking to Scaffold-GS already gives you a 10× reduction (183 → 18.67 MB). This is a strong baseline that many papers don’t even compare against.
  • The context model and hyperprior contribute roughly equally (~18–20% each), and they stack for a 25% combined reduction.
  • PSNR is basically unchanged across all configurations—the compression is essentially “free” in terms of quality.
Step-by-step compression gains. The biggest single jump comes from Scaffold-GS + entropy coding. The context model and hyperprior add meaningful incremental gains on top.

Speed: Not What You’d Expect

“Autoregressive coding” usually means “painfully slow decoding.” In image compression, pixel-level autoregressive models can take minutes to decode a single image. But ContextGS’s autoregressive loop has only 3 iterations (one per level), and within each level, all anchors are coded in parallel. The actual decode time is ~18 seconds on an RTX 3090—comparable to non-autoregressive entropy coding baselines like HAC.

After decompression, rendering speed is identical to (or faster than) Scaffold-GS, because the masking loss tends to reduce the number of active anchors.

Why Not Encode Positions?

An interesting design decision: the authors tried using the hyperprior to entropy-code anchor positions too (marked “w/ APC” in the paper). It works—saves about 1 MB—but the adaptive quantization needed for position encoding hurts PSNR, and the fixed quantization alternative increases the number of symbols so much that encoding/decoding slows down significantly (41s encode vs. 20s without). Since positions are only ~18% of the total, they just store them raw. Pragmatic.

Key Takeaways

  • Context models work in 3D. The idea of predicting not-yet-coded elements from already-coded neighbors transfers cleanly from 2D images to 3D anchor-based representations. The spatial redundancy among anchors is real and exploitable.
  • Level-wise > element-wise autoregression. By operating at 3 hierarchical levels instead of per-element, ContextGS avoids the crippling serial bottleneck of traditional autoregressive models. You get most of the coding gain with none of the speed penalty.
  • Entropy regularization helps quality. A recurring theme in 3DGS compression: constraining the bitrate during training acts as an implicit regularizer, often improving PSNR beyond the uncompressed baseline.
  • Anchor forward is free compression. Reusing coarser-level anchors at the finest level avoids duplicate storage and actually improves quality vs. keeping them separate.
  • Strong baselines matter. Just adding entropy coding + masking to Scaffold-GS already beats most competitors. The context model and hyperprior are incremental-but-meaningful improvements on top of an already-strong baseline.

Things I’d Want to See Next

  • The 3-level hierarchy is fixed. Learning the optimal number of levels and their ratios per scene (or even spatially varying within a scene) could help—flat regions might need fewer levels, detailed areas more.
  • The context model uses only the single nearest coarser-level anchor. Aggregating from multiple neighbors (k-NN context) might capture richer spatial patterns at the cost of more computation.
  • The adaptive quantization step-size \(\Delta\) is learned per channel. More granular (per-block or per-region) quantization could further tighten the rate-distortion curve, similar to what SizeGS does with its ILP-based bit allocation.
  • Integration with hash-grid priors (as in HAC) could be complementary—hash features capture scene-wide patterns, while the context model captures local redundancy.
