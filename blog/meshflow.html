The Core Idea

MeshFlow argues that direct mesh generation does not have to be autoregressive. Existing mesh generators often serialize faces and quantized coordinates into a long token sequence, then predict one token at a time. That choice makes topology easy to express, but it also creates three structural problems: long sequential inference, premature termination, and coordinate quantization error.

The paper replaces that pipeline with a continuous one. A mesh is represented by per-vertex position, outward normal, and a learned edge embedding. Connectivity is recovered by comparing edge embeddings rather than emitting discrete face indices. MeshVAE compresses these continuous vertex tokens, and a rectified-flow Transformer generates all latent tokens in parallel before the decoder reconstructs vertices and topology.

Core idea: make topology differentiable before trying to make generation parallel. MeshFlow first converts discrete adjacency into continuous per-vertex embeddings, then applies the same latent denoising recipe that made parallel image generation practical.

This is worth reading less as another DiT application and more as a representation paper. The flow model is effective because MeshVAE has already turned variable-size geometry and discrete connectivity into a compact continuous sequence. If that latent interface were weak, parallel denoising would only produce invalid edge soups faster.

Problem Setting

The target is an artist-like triangular mesh: relatively compact, explicitly connected, and useful for downstream editing and rendering. The released model takes a surface point cloud or mesh as coarse geometric guidance and can optionally use a reference image. It outputs explicit vertices, normals, and triangular faces rather than an implicit field that must be converted with Marching Cubes.

The central difficulty is the mixed nature of a mesh. Vertex coordinates and normals are continuous; the number of vertices is variable; adjacency and faces are discrete; and face orientation matters. A method must preserve sharp geometry and sparse topology without assuming the result is watertight or manifold.

Indirect 3D generators avoid this problem by generating an SDF, occupancy field, or radiance representation and extracting a mesh afterward. That is effective for shape, but the extracted topology is rarely what an artist would author. It can be excessively dense, smooth sharp edges, or produce an awkward triangulation. Autoregressive methods generate better-structured meshes directly, but their sequence length grows with mesh complexity.

Background

Why Face-Token Autoregression Is Expensive

A naive triangular-face tokenizer repeats three coordinates for each of three face vertices, giving at least \(9n_f\) coordinate tokens for \(n_f\) faces. Better tokenizers exploit indexing and local structure, but generation is still sequential. Long sequences also create a mesh-specific failure mode: if the model emits an end token too early, the geometry is incomplete.

MeshFlow instead starts from vertices. If a mesh has \(n_v\) vertices, its basic representation has \(n_v\) continuous records, and MeshVAE compresses those further to roughly \(n_v/4\) latent tokens. The paper reports 72 times fewer vectors than a naive tokenizer and 16 times fewer than the most efficient compared tokenizer.

Continuous Connectivity

The key device comes from continuous edge representations. For each vertex \(i\), the model predicts an edge embedding \(\mathbf{h}_i\). The main paper writes the topology decoder as a threshold on pairwise embedding distance:

\[ \mathcal{A}_{ij} = \mathbb{I}\!\left[d(\mathbf{h}_i,\mathbf{h}_j) \leq \tau\right]. \]

Here \(\mathcal{A}\) is the recovered adjacency matrix, \(d\) is the learned edge-embedding distance, and \(\tau\) is a threshold. A triangle is then recovered whenever three vertices form a three-edge loop. Predicted vertex normals determine the winding direction and therefore the inside/outside orientation of each face.

Threshold convention: the paper's early representation equation selects edges with \(d\leq\tau\), but its positive-pair loss favors values above \(\tau\). The official code also selects sigmoid(dist - tau) > 0.5, where dist is a signed space-time score. This appears to be a notation mismatch; an implementation should follow the released score definition rather than combine the two inequalities literally.

\[ \mathcal{M} = (\mathbf{V},\mathbf{N},\mathbf{H}), \]

where \(\mathbf{V}\) stores continuous positions, \(\mathbf{N}\) stores outward normals, and \(\mathbf{H}\) stores edge embeddings. This representation does not require the mesh to be watertight or manifold, which is useful for artistic assets that violate clean geometric assumptions.

Rectified Flow

Rectified flow learns a vector field between clean latent samples and Gaussian noise along a straight interpolation path. Unlike autoregressive decoding, the number of denoising steps is fixed independently of the output sequence length, and all mesh latent tokens are updated together at every step.

Method

MeshVAE Input and Output

The encoder receives vertices, normals, and discrete adjacency from a training mesh. Using adjacency in the encoder is not a differentiability problem because it is observed input. The decoder cannot simply emit adjacency, however, so it predicts continuous edge embeddings from which adjacency is reconstructed.

Meshes are padded to a fixed maximum vertex count, and the decoder predicts a validity mask in addition to positions, normals, and edge embeddings:

\[ (\widehat{\mathbf{V}},\widehat{\mathbf{N}}, \widehat{\mathbf{H}},\widehat{\mathbf{m}}) = \mathcal{D}(\mathbf{z}). \]

The mask handles variable vertex counts. The edge embedding handles connectivity. The normals serve two roles: they are geometric attributes and a compact cue for orienting recovered faces.

TokenMerge and TokenSplit

Each input vertex is encoded with Fourier features of its position and normal plus neighboring-vertex information. TokenMerge groups these vertex tokens and applies an MLP, analogous to rearranging spatial channels in pixel shuffle. The merged tokens then cross-attend to the full input sequence, so compression does not discard access to fine geometry.

\[ \mathbf{z} = \operatorname{SA}^{(L_e)}\!\left( \operatorname{CA}(\mathbf{z}_{\mathrm{merged}},\mathbf{X}) \right). \]

The decoder is symmetric. TokenSplit expands the latent sequence back toward the maximum vertex count, and cross-attention to the processed latent code produces the final per-vertex attributes. The ablation results show that this structured initialization is much easier to train than random Q-Former queries or farthest-point-sampled features.

MeshVAE Training Objective

The objective combines valid-vertex classification, coordinate reconstruction, normal reconstruction, adjacency contrast, and KL regularization:

\[ \mathcal{L}_{\mathrm{VAE}} = \mathcal{L}_{\mathrm{mask}} + \mathcal{L}_{\mathbf{v}} + \mathcal{L}_{\mathbf{n}} + \mathcal{L}_{\mathrm{adj}} + \lambda_{\mathrm{kl}}\mathcal{L}_{\mathrm{kl}}. \]

Binary cross-entropy supervises the mask, while MSE is applied to valid positions and normals. Connectivity uses a contrastive objective: connected vertices should fall on one side of the distance threshold and non-connected pairs on the other. Using \(\overline{E}\) for non-edges, the paper's loss is

\[ \begin{aligned} \mathcal{L}_{\mathrm{adj}} &= \mathcal{L}_{\mathrm{pos}} + \lambda_{\mathrm{neg}}R_{\mathrm{neg/pos}} \mathcal{L}_{\mathrm{neg}}, \\ \mathcal{L}_{\mathrm{pos}} &= -\frac{1}{|E|}\sum_{(i,j)\in E} \log \sigma\!\left(d(\widehat{\mathbf{h}}_i, \widehat{\mathbf{h}}_j)-\tau\right), \\ \mathcal{L}_{\mathrm{neg}} &= -\frac{1}{|\overline{E}|}\sum_{(i,j)\in \overline{E}} \log \sigma\!\left(\tau-d(\widehat{\mathbf{h}}_i, \widehat{\mathbf{h}}_j)\right). \end{aligned} \]

The non-edge notation is lightly adapted for readability. These signs reproduce the paper's loss and are consistent with the released code's greater-than score test, but not with the earlier \(d\leq\tau\) equation.

Face Recovery and Boundary Repair

At inference, the decoder mask selects valid vertices, pairwise edge-embedding scores recover adjacency, and three-edge cliques become triangular faces. Average predicted normals determine cyclic order. Because noisy diffusion predictions can leave short boundary loops, the paper describes detecting boundary \(k\)-gons and triangulating loops with \(k<5\).

This post-processing is small but conceptually important. MeshFlow does not guarantee a valid mesh by construction. It predicts a continuous proxy for topology and repairs a narrow class of local failures afterward.

Parallel Rectified-Flow Generation

Let \(\mathbf{x}_0\) be a clean MeshVAE latent and \(\epsilon\) Gaussian noise. The paper uses the linear path

\[ \mathbf{x}_t=(1-t)\mathbf{x}_0+t\epsilon. \]

A DiT predicts the corresponding velocity. With voxelized point-cloud condition \(\mathbf{c}_{\mathrm{vox}}\) injected through 3D rotary position encoding, the conditional flow-matching objective is

\[ \mathcal{L}_{\mathrm{CFM}}(\theta) = \mathbb{E}_{t,\mathbf{x}_0,\epsilon} \left[ \left\| \mathbf{v}_{\theta}\!\left( \operatorname{RoPE3D}(\mathbf{x}_t,\mathbf{c}_{\mathrm{vox}}),t \right) -(\epsilon-\mathbf{x}_0) \right\|_2^2 \right]. \]

The model also receives a target vertex-count signal through the time embedding. During the final training phase, logit-normal time sampling emphasizes difficult fine-detail regions; inference uses a timestep shift of 3.0. The released code defaults to 28 sampling steps.

How Shape Conditioning Enters the DiT

The authors test two approaches. The first samples 32,768 points, encodes them into 2,048 Shape2VecSet-style tokens, and conditions the DiT by cross-attention. It is natural at inference but trains slowly. The second places 3D RoPE directly on vertex coordinates and converges faster, but training vertices and inference-time surface samples follow different distributions.

MeshFlow reduces that domain gap by voxelizing ground-truth vertices during training. A \(32^3\) grid is reported as the best practical balance: coarser spatial support preserves artist-like topology, while higher spatial detail improves shape fidelity.

Experiments

Training Data and Model Scale

The model is trained on an Objaverse-like proprietary collection of roughly 600k professional artist meshes. MeshVAE uses 8-layer Transformer encoder and decoder stacks, hidden width 1,024, and 233M parameters. The main paper describes the DiT as 18 blocks with width 1,024 and 427M parameters.

Reproducibility note: the supplied supplementary TeX describes a different DiT configuration: 16 blocks, width 1,536, and 895M parameters. The released model configuration is access-restricted, so this discrepancy could not be independently resolved from the public checkpoint bundle.

Strengths

Representation-first design

MeshFlow solves the discrete-topology problem before adding diffusion. Positions, normals, and connectivity share one continuous per-vertex interface.

Meaningful efficiency gain

Parallel latent generation removes the long sequential dependency that dominates single-object AR inference, producing a large practical speedup.

Compact without quantization

MeshVAE compresses far more aggressively than token-based encoders while avoiding coordinate bins that can collapse vertices and faces.

Explicit downstream asset

The output is a sparse editable mesh with vertices and faces, not an implicit surface that still needs topology extraction and cleanup.

Limitations

Main limitation: MeshFlow generates only triangular, untextured meshes and still relies on heuristic repair when noisy edge predictions create small holes.

The first limitation is topology coverage. Artists often prefer quad-dominant topology for subdivision, rigging, and deformation. MeshFlow recovers triangles from three-edge cliques, so supporting quads or mixed polygons would require a different face-recovery model rather than a small decoder change.

The second limitation is validity. The continuous edge representation makes topology learnable, but it does not guarantee a valid face complex. The method triangulates short boundary loops as post-processing, and the supplementary shows failure cases with holes caused by noisy positions and edge embeddings. Three-edge clique recovery can also be brittle when adjacency errors create or delete a single edge.

The third limitation is evaluation. CD and HD strongly reward geometric proximity but say little about face quality, flipped normals, non-manifold structure, holes, edge flow, or editability. The authors acknowledge this directly. A convincing artistic-mesh benchmark still needs topology-aware and workflow-aware evaluation.

The fourth limitation is reproducibility. Training uses approximately 600k proprietary professional assets, so dataset quality and filtering cannot be reproduced from public data. The architecture discrepancy between the main paper and supplementary also needs clarification. Finally, the released pipeline still expects coarse surface geometry; image conditioning is optional rather than a replacement for point-cloud guidance.

Texture generation is outside the scope. There is no UV layout, material assignment, or texture synthesis, so MeshFlow addresses geometry authoring rather than a complete production-ready asset pipeline.

Research Insight

Research insight: parallel mesh generation becomes practical when topology is represented as a continuous relation among vertices, not when a diffusion model is applied directly to face indices.

The deeper pattern is familiar from successful generative systems: spend representational effort to create a smooth latent space, then use a simple scalable generator. MeshFlow's strongest contribution is a latent interface in which geometry and topology can be denoised together. Rectified flow is then a natural consequence rather than the conceptual starting point.

The paper also exposes a useful division of labor. Normals are not only rendered attributes; they remove the need to predict a more complicated local edge-order representation. Edge embeddings handle adjacency, normals handle orientation, and the validity mask handles variable cardinality. Each continuous channel absorbs one discrete bookkeeping problem.

I would qualify the paper's linear-scaling language. MeshFlow removes \(O(N)\) sequential decoding steps, which is the main practical speed advantage. But a standard Transformer still performs self-attention over the latent sequence, so its per-layer compute is not literally linear in token count. The defensible claim is fixed-step parallel generation with much better wall-clock scaling than token-by-token AR decoding.

Practical Perspective

The released pipeline is useful for coarse-geometry-to-clean-mesh workflows. A surface point cloud establishes shape; an optional reference image supplies appearance-related semantic cues through DINOv3 features; the output can be exported as GLB. The repository exposes a target vertex-count control, although it should be understood as a coarse conditioning signal rather than exact polygon budgeting.

For a research implementation, the risky component is not the DiT loop. It is robustly preparing and decoding mesh structure: duplicate removal, degree filtering, padded vertex sampling, neighborhood features, threshold calibration, clique extraction, orientation, and boundary repair. Reproducing the claimed quality requires treating these geometry operations as part of the model, not as incidental utilities.

For games and VFX, MeshFlow is closer to a useful geometric draft than most implicit generators. It produces compact explicit topology quickly. It still needs retopology judgment, UVs, materials, and failure inspection before production use, especially for deformable characters or assets requiring quad flow.

Future Directions

  • Quad and mixed-polygon generation: replace triangle-clique recovery with a representation that supports explicit polygon loops and artist-controlled edge flow.
  • Validity by construction: incorporate differentiable constraints or structured decoding that prevents holes, duplicate faces, and non-manifold adjacency rather than repairing them afterward.
  • Topology-aware metrics: evaluate manifoldness, boundary defects, valence distributions, normal orientation, editability, and deformation behavior alongside geometric distance.
  • Public-data replication: measure how much performance comes from the representation versus the proprietary professional-asset distribution.
  • Image-first generation: remove the requirement for coarse point-cloud guidance while preserving explicit artist-like topology.
  • Joint UV and material generation: extend the latent representation to seams, UV charts, texture coordinates, and PBR attributes.
  • Scalable attention: use sparse, local, or hierarchical attention so parallel generation remains efficient as vertex budgets move well beyond 8K.

Key Takeaways

  • MeshFlow is primarily a representation contribution. Continuous edge embeddings make discrete topology compatible with VAE compression and flow generation.
  • TokenMerge is essential. The ablation gap shows that structured latent initialization matters far more than simply reducing the token count.
  • Parallel generation delivers a real speed advantage. The reported single-object runtime is about 1.2 seconds versus 21 seconds for the fastest AR baseline.
  • Geometry metrics favor the method but remain incomplete. CD and HD do not evaluate the topology qualities implied by the term "artist-like."
  • The output is useful but not complete. MeshFlow produces explicit triangular geometry; quads, guaranteed validity, UVs, and textures remain open problems.
