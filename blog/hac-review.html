[Paper Review] HAC: Hash-grid Assisted Context for 3D Gaussian Splatting Compression

Venue

The Compression Gap

Scaffold-GS made 3DGS more structured by introducing anchor points that spawn neural Gaussians via MLPs. But even with this improvement, each anchor is still treated independently. The model stores per-anchor features (\(\mathbf{f}_a \in \mathbb{R}^{D_a}\)), scaling (\(\mathbf{l} \in \mathbb{R}^6\)), and offsets (\(\mathbf{o} \in \mathbb{R}^{3K}\)) without leveraging the fact that neighboring anchors on the same surface tend to have very similar attributes.

Previous 3DGS compression methods focused on “values”—pruning unimportant Gaussians and quantizing parameters via codebooks. These are necessary, but they ignore structural redundancy: the spatial correlations between nearby primitives. In image and video compression, exploiting such structure is what makes the difference between naive JPEG and modern learned codecs. HAC brings this same principle to 3DGS.

The Core Idea: Using a Hash Grid as Context

HAC’s insight is beautifully simple. Instead of compressing anchor attributes in isolation, use a jointly-learned hash grid to predict what each anchor’s attributes should be, and then only entropy-code the residual surprise. If the prediction is good (high probability), the attribute takes very few bits. If it’s off (low probability), it costs more. This is exactly how context models work in image compression.

The key question the authors had to answer: Is there actually mutual information between the hash grid and anchor attributes? They verified this by a simple experiment—substituting anchor features \(\mathbf{f}_a\) with interpolated hash features \(\mathbf{f}_h\). The result: only moderate quality drop (33.41 → 32.85 PSNR on Synthetic-NeRF), confirming that the hash grid captures most of the information that anchor features encode.

But instead of replacing anchor features (which hurts rendering quality and destabilizes training), HAC uses hash features as context for probability estimation during entropy coding. This is a critical distinction: the actual rendering pipeline stays identical to Scaffold-GS. The hash grid is only needed during encoding/decoding, not at inference time.

The elegant part: Since the hash grid is only used for compression (not rendering), the rendering speed and quality upper-bound are completely unaffected. After decoding, you throw away the hash grid and run standard Scaffold-GS.

How It Works

The Probability Framework

HAC frames compression as conditional probability estimation. For the set of anchor attributes \(\mathcal{A} = \{\mathbf{f}_a, \mathbf{l}, \mathbf{o}\}\):

\[ p(\mathcal{A}, \mathbf{x}_a, \mathcal{H}) = p(\mathcal{A} | \mathbf{x}_a, \mathcal{H}) \times p(\mathbf{x}_a, \mathcal{H}) \approx p(\mathcal{A} | \mathbf{f}_h) \times p(\mathcal{H}) \]

where \(\mathbf{f}_h = \text{Interp}(\mathbf{x}_a, \mathcal{H})\) is the hash feature interpolated at anchor location \(\mathbf{x}_a\). Higher \(p(\mathcal{A} | \mathbf{f}_h)\) means fewer bits. The hash grid itself (\(p(\mathcal{H})\)) also needs to be stored, but it’s binarized to keep it tiny.

The Context Model

For each anchor, the context model works in three steps:

The three-step context model: query the hash grid for spatial context, predict quantization refinement and distribution parameters via a shared MLP, then entropy-code anchor attributes using the predicted Gaussian distributions.

The empirical observation that anchor attributes follow Gaussian distributions is what makes this work cleanly. The context MLP only needs to predict two parameters (\(\boldsymbol{\mu}, \boldsymbol{\sigma}\)) per attribute dimension, and the probability integral becomes a simple CDF difference.

Adaptive Quantization Module (AQM)

A subtle but important detail: different attributes need different quantization step sizes. Anchor features \(\mathbf{f}_a\) are high-dimensional but have integer-like magnitudes. Scaling \(\mathbf{l}\) and offsets \(\mathbf{o}\) are small decimals. Using a single quantization step would destroy the small-valued attributes.

AQM solves this with per-anchor, per-attribute adaptive step sizes:

\[ \mathbf{q}_i = Q_0 \times (1 + \tanh(\mathbf{r}_i)), \quad \mathbf{r}_i = \text{MLP}_q(\mathbf{f}^h_i) \]

where \(Q_0\) is a base step size (1.0 for features, 0.001 for scaling, 0.2 for offsets) and \(\mathbf{r}_i\) is a learned refinement. The \(\tanh\) bounds the step to \((0, 2Q_0)\), so each anchor can get a finer or coarser quantization depending on what the hash grid context suggests.

Binary Hash Grid

The hash grid itself needs to be compact, since it goes into the bitstream. HAC binarizes all hash parameters to \(\{-1, +1\}\) using straight-through estimation. The grid uses a mixed 3D-2D structure: 12 levels of 3D embeddings (resolution 16–512) plus 4 levels of 2D embeddings (resolution 128–1024), each with feature dimension 4. Since the grid is binary, it’s compressed with simple frequency-based arithmetic coding.

Adaptive Masking

The authors observed that many offsets \(\mathbf{o}\) cluster at zero—meaning many spawned Gaussians are unnecessary. HAC applies learned binary masks (via straight-through estimation) to prune these. If all offsets of an anchor get masked, the entire anchor is pruned—saving its position, features, and everything else.

The Training Pipeline

Training is carefully staged to avoid instability:

Three-phase training: warm up Scaffold-GS, introduce quantization noise, then activate the full HAC framework with hash grid and entropy constraints. Bar widths are proportional to iteration count.

The overall loss combines three components:

\[ \mathcal{L} = \mathcal{L}_\text{Scaffold} + \frac{\lambda_e}{N(D_a + 6 + 3K)}(\mathcal{L}_\text{entropy} + \mathcal{L}_\text{hash}) + \lambda_m \mathcal{L}_m \]

where \(\mathcal{L}_\text{entropy}\) measures the bit cost of anchor attributes, \(\mathcal{L}_\text{hash}\) measures the hash grid’s bit cost, and \(\mathcal{L}_m\) encourages masking of redundant offsets. The parameter \(\lambda_e\) controls the rate–distortion tradeoff: larger \(\lambda_e\) means more aggressive compression at the cost of some quality.

What Goes Into the Bitstream?

The final compressed representation has five components:

Anchor attributes dominate the bitstream. The hash grid, masks, and MLPs together add less than 1 MB of overhead. Positions are stored at 16-bit precision without entropy coding.

The hash grid overhead is only ~0.15 MB—because it’s binary and compressed. The MLPs are 0.16 MB (stored at full precision since they’re tiny and shared). The dominant cost is the entropy-coded anchor attributes, which is exactly what the context model is designed to minimize.

HAC achieves this while matching or exceeding Scaffold-GS in PSNR. For instance on Mip-NeRF 360, HAC-highrate gets 27.77 dB PSNR (vs. Scaffold-GS’s 27.50) at 21.9 MB (vs. 253.9 MB). The fidelity improvement comes from two sources: (1) the entropy loss acts as a regularizer that prevents overfitting, and (2) the anchor feature dimension is increased to 50 (from 32 in Scaffold-GS), providing more capacity that the compression scheme can efficiently handle.

An important practical detail: rendering speed is preserved. Since the hash grid is discarded after decoding, HAC runs at the same FPS as Scaffold-GS (actually slightly faster due to pruned Gaussians). On BungeeNeRF: 283 FPS for HAC vs. 232 FPS for Scaffold-GS.

Ablation: What Matters Most?

The paper provides clean ablations by disabling each component:

  • Without hash grid context (setting \(\mathcal{H} = 0\)) — The probability drops from \(p(\mathcal{A} | \mathbf{f}_h)\) to \(p(\mathcal{A})\). Bit consumption drastically increases because the model can only use global statistics (a single \(\mu, \sigma\) per attribute type) instead of per-anchor predictions. This is the single biggest contributor to compression performance.
  • Without AQM — Using only the base step \(Q_0\) without per-anchor refinement. Significant fidelity drop, especially on complex scenes at higher bitrates. The fixed quantization is too coarse for some attributes and too fine for others.
  • Without offset masking — Rate savings decrease, especially in simpler scenes where many Gaussians are genuinely unnecessary. The masking is most impactful at lower bitrates.

Bit Allocation Visualization

One of the more insightful analyses in the paper is the bit allocation visualization. The authors voxelize the 3D space and color each voxel by its total (or per-anchor average) bit consumption:

  • Total bits per voxel: Complex regions (specular objects, sharp edges) consume more total bits, as expected—they need more anchors with harder-to-predict attributes.
  • Per-anchor bits: Interestingly, per-anchor bit consumption is smooth across space. In high-complexity regions, the model creates more anchors (increasing total bits) but each individual anchor’s attributes are well-predicted by the hash grid context (keeping per-anchor bits low). This confirms the assumption that nearby anchors share inherent consistency.

This is elegant: the model naturally adapts bit allocation by adjusting both how many anchors exist in a region (via masking/pruning) and how many bits each anchor needs (via context-conditioned probability). The two mechanisms complement each other.

Position in the Compression Landscape

HAC occupies a very specific position in the 3DGS compression timeline:

  • Before HAC: Compression was “value-based”—pruning, codebooks, vector quantization. Each anchor/Gaussian was treated independently.
  • HAC: Introduced context modeling to 3DGS compression. The hash grid serves as the “spatial context” that bridges unorganized anchors.
  • After HAC: ContextGS took the context idea further with hierarchical autoregressive prediction across octree levels. SizeGS explored RAHT transforms for another way to exploit spatial correlations.

What’s interesting is the contrast between HAC and ContextGS in their approach to context. HAC uses a separate data structure (the hash grid) as context. ContextGS uses the anchors themselves as context for finer-level anchors (autoregressive). Both are valid approaches inspired by different traditions: HAC draws from Instant-NGP/learned image compression, while ContextGS draws from progressive coding/autoregressive models.

Key Takeaways

  • Context modeling is the real game-changer. Going from no context (\(p(\mathcal{A})\)) to hash-grid context (\(p(\mathcal{A} | \mathbf{f}_h)\)) accounts for the majority of the compression gain. This is the same lesson that image compression learned: entropy coding without a good probability model is just a fast baseline.
  • Separation of concerns is powerful. Using the hash grid only for compression (not rendering) means you get compression for free—no rendering overhead, no quality ceiling changes, and the grid can be discarded after decoding.
  • Adaptive quantization matters more than you’d think. AQM’s per-anchor, context-conditioned quantization step is a small change that makes a large difference, especially for the multi-scale attributes (features vs. scaling vs. offsets).
  • Binary hash grids are extremely cheap. The entire multi-resolution hash grid compresses to ~0.15 MB. Binarization via STE is aggressive but sufficient for context prediction, where you don’t need high-precision features—just enough mutual information with the anchors.
  • The staged training is non-trivial. The paper’s careful three-phase schedule (pure Scaffold-GS → quantization adaptation → full HAC) is likely essential for convergence. Jointly training everything from scratch would probably not work, as quantization noise destabilizes anchor spawning early on.
